nettiers 1.0.0 quick start

Download beta 1 
			


requirements
	
  • Enterprise library june 2005 is requiered.
	
  • Codesmith 2.6+ (much much faster with codesmith 3)
	
  • nUnit 2 for unit tests (optional)
	
  • nAnt (optional)
	
  • nDoc (optional)
	
  • Enterprise Library for ASP.NET 2.0 (http://www.rationalpath.com/)
	







    1. INSTALLATION
	- run the install of the NetTiers msi
	- next, next and next ;-)




    2. GENERATE THE ".netTiers"

- in "start menu / programs / nettiers" click on the nettiers icon to lauch the templates in codesmith explorer

- On the left pane, fill the properties. 
	The most important ones to start are:
	
	[DataSource]
		- EntireDatabase: indicates whether or not all the tables and views will be included in the generation.
		- SourceDatabase: the database from which you want to generate your "nettiers"
	
	[General]
		- NameSpace: the root namespace for the generated code. the "xxxFolderName" are used to build the underlying namespace and the sub folders hierarchy as well.
		- OuputDirectory: the output folder
		- IncludeUnitTest: indicates if the unit tests will be generated.
		- VsNetIntegration: indicates the kind of visual studio projects output is wanted.
		- VsNetVersion: indicates the version of visual studio

	

			

    3. CREATE YOUR APPLICATION

	- launch the generated solution
	- add a winform or webform application
	- add references to the generated project
	- add code that consume the .netTiers

			here is a sample code to help you start:
			using myDb.DataAccessLayer;
			...
			
			ProductCollection products = DataRespository.ProductProvider.GetByCategoryId (123);
			foreach(Product product in products)
			{
			Console.WriteLine(product);
			}
			
	- add a app.config or web.config to your application

	


    4. CONFIGURATION
	- Start the enterprise library configuration console
	- open the *.config  file of your application
	- right click on the application and select the '.nettiers configuration' menu item
	- adjust the database instance and connection string
	- back in vs.net, set a PostBuildEvent to your application so the config files will be copied to the output folder: 
	
		copy "$(ProjectDir)*.config" "$(TargetDir).

	
 

    5. RUN
